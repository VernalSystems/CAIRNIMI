What is SIREN?

A revolutionary spatial indexing system that encodes geographic and abstract locations into a concise, semantically meaningful format.

Recursive Tiling

SIREN subdivides Earth through recursive grid patterns, alternating between 8×8 and 4×4 subdivisions for efficient spatial indexing at multiple scales.

Semantic Naming

Unlike traditional coordinates, SIREN uses structured vocabulary to create location identifiers that are both human-readable and machine-processable.

Vector & Time Encoding

SIREN integrates temporal indexing and vector shapes into its format, allowing for the representation of dynamic regions and time-bound spatial entities.

Hex Symbol Codes

Each SIREN code can be paired with a compact hexagonal visual representation, ideal for scannable markers, augmented reality, and visual documentation.

SIREN vs. Google Plus Codes

A deep dive into the systemic, computational, and architectural differences between SIREN and Plus Codes (OLC).

Aspect SIREN Google Plus Codes
Code Structure Semantically layered, symbolic names + vector/time metadata Alphanumeric grid codes derived from lat/long
Shortened Code Uniqueness Globally unique without needing place names Require locality info to disambiguate
Data Dependency Self-contained, not reliant on external locality datasets Sensitive to changes in locality/place name datasets
Point-level Precision Supports sub-meter precision with offsets Area-based; doesn't uniquely identify units inside buildings
Grid System Alignment Adaptable tiling; semantically anchored tiles Fixed WGS84 grid; may misalign with property or administrative lines
Computation & Proximity Direct proximity analysis from encoded structure Requires full decoding for spatial comparison
AI Integration Symbolic and vector-compatible, ideal for AI parsing Limited semantic meaning; not structured for symbolic AI
Infrastructure Dependency Open and portable with no central dependency Effectively tied to Google Maps for usability

SIREN vs. what3words

A detailed technical comparison of SIREN and what3words reveals key design strengths and limitations in symbolic geocoding systems.

Aspect SIREN what3words
Transparency & Openness Open semantics and structure; vendor-independent Proprietary and closed-source; vendor lock-in
Address Confusion Resistance Created with semantic anchoring, optional checks Confusable nearby addresses; no inherent error correction
Error Correction Capabilities Optional metadata and vector correction fields No checksums or intrinsic error detection
Word Semantics Layered and semantic word composition Words are arbitrary and semantically empty
Spatial Resolution Multiscale: from region to sub-meter with offset Fixed 3×3 meter grid; no hierarchy or scale
Offline/Independent Use Fully self-contained and interpretable API-dependent for suggestions and validation
Vector & Time Support Includes vector shapes and temporal data Lacks built-in vector or temporal encoding
Legal and Ecosystem Flexibility No legal restriction on independent implementations Legal threats to open implementations; no ecosystem freedom

Code Structure

SIREN's powerful format combines location, time, and shape information in a single, versatile reference.

SILVERON::COVE | +3 -2 +0 @2025-05-05T00:00Z #Line !{Future use}
Tile Name
SILVERON::COVE
Semantic identifier for the spatial tile, drawn from a structured vocabulary.
XYZ Offset
+3 -2 +0
Positional offsets in meters from the tile center (X=east-west, Y=north-south, Z=vertical).
Timestamp
@2025-05-05T00:00Z
ISO 8601 timestamp for capturing temporal data or for time-sensitive locations.
Vector
#Line
Vector shape or geometric region associated with the location.
Reserved
!{Future use}
Reserved for future mathematical expressions or metadata tags.
